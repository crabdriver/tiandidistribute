# Ordo Desktop Workbench MVP Manual Validation

## 目标

验证 `desktop/` 工作台已经打通：

- 微信配置可视化填写
- 导入内容
- 模板 / 封面配置
- 发起草稿 / 正式发布
- 日志 / 最近结果回看
- 头条号任务级定时发布时间

## 前置条件

1. 仓库根目录已安装 Python 依赖：

```bash
python3 -m pip install -r requirements.txt
```

2. `desktop/` 已安装前端依赖：

```bash
cd desktop
npm install
```

3. 浏览器平台验证前，先打开启用远程调试的 Chrome，并确认知乎 / 头条号 / 一点号已登录。

4. 若本机 Python 不是 `python3`，先设置：

```bash
export ORDO_PYTHON=/path/to/python
```

## 启动

```bash
cd desktop
npm run tauri:dev
```

## 验证步骤

### 1. 工作台启动

- 窗口标题显示为 `Ordo Workbench`
- 首页顶部可见 `设置` 按钮与微信配置状态
- 首页可见三栏布局：文章列表 / 模板与预览 / 封面与执行区
- 底部可见日志区

### 2. 微信设置

- 点击顶部 `设置`
- 输入 `AppID`、`Secret`、`Author`
- 点击 `保存设置`
- 预期：
  - 弹窗关闭
  - 顶部状态从 `微信配置未完成` 变为 `微信配置已就绪`
  - 底部日志或状态区出现保存成功反馈

### 3. 粘贴导入

- 点击 `粘贴导入`
- 输入测试正文
- 点击 `导入`
- 预期：
  - 左侧出现文章卡片
  - 中间预览区出现规范化 Markdown
  - 右侧出现平台与封面分配区

### 4. 单文件 / 文件夹导入

- 点击 `单文件` 选择一篇 `.md` 或 `.txt`
- 点击 `文件夹` 选择含多篇文章的目录
- 预期：
  - 能正确生成文章列表
  - 每篇文章都带 `article_id`
  - 主题池 / 封面池状态在右侧正确显示

### 5. 模板与封面配置

- 在 `全局模板模式` 中切换 `默认随机` / `逐篇自定义`
- 为当前文章切换主题
- 为知乎 / 头条号 / 一点号手动覆盖封面
- 预期：
  - 配置变更后不会报错
  - 执行区任务统计会跟着刷新

### 6. 草稿发布链路

- 将模式保持在 `存草稿`
- 点击 `开始发布`
- 预期：
  - 若已勾选 `wechat` 且配置缺失，执行区会先显示明确阻塞提示
  - 底部日志按顺序出现 `job_started`、`article_started`、`platform_started`、`platform_finished`、`job_finished`
  - 完成后执行区统计会刷新
  - 最近结果区能看到新记录

### 7. 平台专项检查

- 知乎：发布设置中的创作声明应为 `内容包含AI辅助创作`
- 头条号：展示封面应切到 `单图`，作品声明应勾选 `引用AI`
- 一点号：封面应切到 `单图`，内容声明应为 `内容由AI生成`

### 8. 正式发布与头条定时发布

- 将模式切到 `正式发布`
- 勾选 `知乎 / 头条号 / 一点号 / 简书`
- 预期：
  - 头条号被选中时，执行区会出现 `datetime-local` 的定时发布时间输入
  - 未填写该时间时，头条号走普通正式发布
  - 填写该时间时，bridge 规划结果中的头条上下文应包含 `scheduled_publish_at`
  - 头条号成功预约后，结果状态应为 `scheduled`，而不是普通 `published`
  - 简书在正式发布模式下不应分配封面，也不应要求 AI 声明

## 当前已知边界

- 简书自定义封面仍保持诊断式失败，不视为稳定自动化能力
- 浏览器平台依赖目标站点 DOM；若平台改版，需要重新校对选择器
- 当前桌面 MVP 默认从仓库源码目录调用 Python，不涉及最终安装包内嵌 Python
