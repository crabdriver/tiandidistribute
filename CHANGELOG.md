## v0.2.0

发布日期：2026-03-15

本次版本完成了从单一微信公众号草稿工具到多平台发布工作流的升级，并补齐了版本记录。

### 新增

- 新增统一入口 `publish.py`，支持按单篇或目录批量发布。
- 新增 `zhihu_publisher.py`、`toutiao_publisher.py`、`jianshu_publisher.py`、`yidian_publisher.py`。
- 新增 `live_cdp.mjs` 长驻 broker 机制，复用 Chrome CDP 连接，减少重复授权。
- 新增 `BATCH_PUBLISHING_GUIDE.md`，整理批量发布、标题清洗、封面要求和限制说明。
- 新增 `markdown_utils.py`，统一 Markdown 转 HTML、纯文本和源文格式处理。
- 新增 `VERSION` 与本变更日志文件，用于后续版本发布记录。

### 改进

- 微信支持通过命令行按单篇文章发布草稿或正式发布。
- 微信正文、摘要与其它平台统一走 Markdown 解析链路。
- 微信自动识别中文编号段落（如 `1、2、3、`），避免多个条目挤成同一大段。
- 知乎正文改为更稳定的纯文本注入，去除 Markdown 符号外露问题，并修复段落被拆散的问题。
- 头条号与一点号正文改为注入真正的 HTML 结构，保留标题、粗体、段落等格式。
- 简书改为写入规范 Markdown 源文，避免将伪 HTML 直接塞入编辑器。
- 所有平台统一移除文章标题中的数字前缀，如 `11-03_`。
- `publish.py` 增加工作台绑定、自动开页、自动预热、预检查、结果分类与记录。

### 验证

- 简书当前能完成正文写入，但正式发布受平台每日公开文章上限影响。
